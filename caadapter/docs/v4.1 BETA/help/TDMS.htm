Introduction

This chapter describes caAdapters mapping capability from a CSV metadata to an object model, or a data model. It provides step-by-step instructions to create a CSV metadata file from a CSV data file, export a UML model from EA to an XMI file, and map a CSV metadata to the object model, or data model, represented in the XMI file.

Create a CSV Metadata File

A CSV metadata file defines the specification of a comma-separated data file. caAdapter supports two kinds of CSV metadata: structural and non-structural. The structural metadata presents the segment-based structure of CVS data files. Each segment may contain zero or more children segments, and zero or more fields. The non-structural metadata is a simplified version of the structural metadata. There is only one segment, which is the root segment, with a series of children fields.

 

The following steps describe how to create a new non-structural metadata file from an existing CSV data file:

 

1.    Select File > New > CSV Specification (Figure 8‑1)

 

Figure 8‑1 New CSV Specification Menu Item

1.    The system will display a selection dialog box. Check the Generate from a Non-Structure CSV file (Figure 8‑2).

 

Figure 8‑2 Select Data Source to Create New CSV Metadata

 

2.    Click the Browse button to locate the source CVS data file (Figure 8‑3).

 

Figure 8‑3 Locate the Source CSV Data File

3.    Once the file is selected, caAdapter will display the selected CVS data file name and path (Figure 8‑4).

 

Figure 8‑4 File Name and Path of the Selected CSV Data File

4.    Click OK to start the generation process. The system will then present the generated non-structure CSV metadata in the CSV specification panel (Figure 8‑5).

Figure 8‑5 Created CSV Metadata File from a CSV Data File

Export a UML Model to an XMI File

Before beginning to map between CSV data elements to an object model or a data model, the user must create an XMI file containing a representation of both models. The user can use EA to create this XMI file using the following steps:

 

1.     Open the .eap file that contains the object and data models) and right click on Logical View. Select Import/Export > Export package to XMI file (Figure 8‑6).

 

Figure 8‑6 Export XMI file from EA

2.     On the Export Package to XMI window, check the Format XMI Output box and the Enable Full EA Roundtrip box. Select XMI Type as UML1.3(XMI 1.1).  Specify the output file name of the XMI file and click Export to create the XMI file (Figure 8‑7).

Figure 8‑7 Options to export XMI file from EA

Note: Make sure to uncheck Use DTD option as shown with Figure 2-7. If the Use DTD option is checked, the generated XMI file will contain the DOCTYPE tag: <!DOCTYPE XMI SYSTEM "UML_EA.DTD">. This will cause the caAdapter XMI loader to verify Document Type Definition (DTD) and may cause validation errors if the required DTD (UML_EA.DTD) is not available.

Map a CSV Metadata to an Object Model or Data Model

The following steps will guide you through mapping a CSV data elements to the object, or data, model.

1.    Select File > New > CSV Meta To Object/Data Model Mapping (Figure 8‑8).

Figure 8‑8 Invoke new CSV Meta to Object Model Mapping menu

2.    The system will prompt you to specify the mapping target; i.e. Object Model or Data Model (Figure 8‑9)

Figure 8‑9 Select Mapping Target Model

3.    The system will then present an empty mapping panel (Figure 8‑10).

Figure 8‑10 Empty Mapping Panel

4.    Click Open CSV Meta File button to display the Open CSV Meta file  dialog box (Figure 8‑11). Select the source CSV metadata file.

Figure 8‑11 Select Source CSV Metadata File

5.    Once a valid CSV metadata file is selected, the system will load that in the left panel of the mapping tool in a tree structure (Figure 8‑12).

Figure 8‑12 Loaded CSV Metadata into Mapping tool

6. Click Open XMI File button to select the target XMI file. The Open CSV XMI file  dialog box will be opened (Figure 8‑13). Select the target XMI file.

Figure 8‑13 Select Target XMI File

7. Once a valid XMI file is selected, the system will then load that in the right panel of the mapping tool. The selection in Step 2 above will determine whether caAdapter will load the Object Model or the Data Model in the right panel (Figure 8‑14 and Figure 8‑15).

Figure 8‑14 Loaded Object Model in the Mapping Tool

Figure 8‑15 Loaded Data Model in the Mapping Tool

Note: Steps 4,5 and Steps 6,7 are exchangeable. The user may load either the CSV metadata file or the XMI file first. caAdapter in neutral to such order.

8.     Once the source and destination files have been selected displayed in the mapping tool, the user is now ready to perform the mapping using drag-and-drop mechanism Figure 8‑16).

 

Figure 8‑16 Mapping Source to Target Using Drag-and-Drop

Note: This version of the TDMS Module uses the following basic mapping rules. Additional mapping rules may be implemented in later versions.

a.     The user may not map from a CSV segment. Mapping can only be done from  a CSV field within the segment

b.     The user may not map to a table, or to an object. The target mapping can only be a table column, or an object attribute

c.     The user may map multiple CSV fields to the same target field. A target field can be a table column or an object attribute

d.     The same CSV field may be mapped to more than one target field. A target field can be a table column or an object attribute

 

Validate the Mapping

The user can invoke the validation process in two ways: select File > Validation from the menu bar; or click the validation  icon on the toolbar. Figure Figure 8‑17 shows the validation report. The validation report is presented in an XML format. The user may save that for later use.

 

Note:   The current validation process is limited to identifying unmapped fields in the metadata file. The validation report contains a list of those fields. Additional validation rules may be defined and implemented in later versions.
